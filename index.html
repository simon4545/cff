开始使用C.F.F

C.F.F提供了一套完整的前端开发及编译，部署的方案。借助C.F.F的Mock机制，前端开发人员可以完全脱离后台环境开发，并可以在项目集成时无缝切换到真实的数据环境。

安装C.F.F

C.F.F是完全免安装、免前置条件的，可以在Windos/Linux/Mac系统下全功能运行。但目前项目并未放入公网中，如有需要请联系xlzhou2@iflytek.com

运行C.F.F

只需要下列简单的4步就可以在Linux/Window/Mac环境下成功运行C.F.F

  1. 将C.F.F的安装包解压至网站任一级目录下(推荐网站根目录,假定Windows下为D:\\www，Linux/Mac下为/var/www)
  2. 进入D:\\www\\Build目录
  3. 运行C.F.F网页服务器
    • Linux环境:执行node /var/www/Build/service.js
    • Windows环境:双击运行D:\\www\\Build\\dev.bat
  4. 打开浏览器访问http://127.0.0.1:8006

开发一个案例网站

从Hello World开始

  • Linux环境:进入/var/www/Module目录
  • Windows环境:进入D:\\www\\Module目录
  1. 在当前目录下,新建HelloWorld目录，并在该目录下新建index.html,index.js(非必须),index.css(非必须)
  2. 在index.html文件中写入html代码 
                            <!DOCTYPE html>
                        <html>
                        <head>
                        <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
                        <title>
                        知识点选题-梯子网
                        </title>
                        </head>
                        <body>
                        Hello World!
                        </body>
                        </html>
  3. 访问http://127.0.0.1:8006/Module/HelloWorld/
到这里，一个简单的HelloWorld页面就建成了，访问试试吧!

基于Smarty模板+模拟数据的动态网页

C.F.F内嵌Smarty语法支持，所以不需要做额外的工作，就可以很轻松的基于Smarty模板开发网站,并可以轻松的创建模拟后端假数据。

  • Linux环境:进入/var/www/Module/HelloWorld目录
  • Windows环境:进入D:\\www\\Module\\HelloWorld目录
  1. 修改index.html文件如下: 
                            <!DOCTYPE html>
                        <html>
                        <head>
                        <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
                        <title>
                        知识点选题-梯子网
                        </title>
                        </head>
                        <body>
                        Hello {literal}{$who},I'm {$age}{/literal} years old!
                        </body>
                        </html>
  2. 在当前目录创建data.json文件，内容如下: 
                            {
                            "who":"World",
                            "age":"23"
                        }
  3. 访问http://127.0.0.1:8006/Module/HelloWorld/
到这里，Smarty模板并集成假数据的页面就建成了!C.F.F实现了Smarty 90%语法支持，还有很多标签可以使用...前端调试后端数据就是这么轻松!

基于AJAX+模拟数据的动态网页

C.F.F支持正则表达式或字符串匹配两种方式创建模拟后端假数据。

  • Linux环境:进入/var/www/Module/HelloWorld目录
  • Windows环境:进入D:\\www\\Module\\HelloWorld目录
  1. 修改index.html文件如下: 
                            <!DOCTYPE html>
                        <html>
                        <head>
                        <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
                        <title>
                        知识点选题-梯子网
                        </title>
                        <script type="text/javascript" src="http://lib.sinaapp.com/js/jquery/1.7.2/jquery.min.js"></script>
                        </head>
                        <body>
                        Hello {literal}{$who},I'm {$age}{/literal} years old!
                        <script type="text/javascript">
                            $.get('/index.php?app=resource&mod=Homepage&act=index',function(result){
                                $("body").html(result);
                            });
                        </script>
                        </body>
                        </html>
  2. 在当前目录创建route.json文件(如果已有，则不必重复创建)，内容如下: 
                            [
                            {"key":"/index.php?app=resource&mod=Homepage&act=index","value":"data.json"}
                        ]
  3. 访问http://127.0.0.1:8006/Module/HelloWorld/,AJAX请求/index.php?app=resource&mod=Homepage&act=index,则会指向到
    • Linux环境下的/var/www/Module/HelloWorld/data.json文件
    • Windows环境:进入D:\\www\\Module\\HelloWorld\\data.json文件
route.json支持多条解析，我们可以结合正则实现按传入参数加载不同的Mock数据
  1. 修改route.json文件如下: 
                            [
                            {"key":"/index.php?app=resource&mod=Homepage&act=index","value":"data.json"},
                            {"key":/\\/index\\.php\\?app=resource&mod=(\w+)&act=(\d+)/i,"value":"data_$1_$2.json"}
                        ]
  2. C.F.F路由引擎会把正则匹配的分组自动替换到$1,$2,$n...
  3. 修改index.html文件如下: 
                            <!DOCTYPE html>
                        <html>
                        <head>
                        <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
                        <title>
                        知识点选题-梯子网
                        </title>
                        <script type="text/javascript" src="http://lib.sinaapp.com/js/jquery/1.7.2/jquery.min.js"></script>
                        </head>
                        <body>
                        Hello {literal}{$who},I'm {$age}{/literal} years old!
                        <script type="text/javascript">
                            $.get('/index.php?app=resource&mod=test&act=212',function(result){
                                $("body").html(result);
                            });
                        </script>
                        </body>
                        </html>
  4. 再次访问http://127.0.0.1:8006/Module/HelloWorld/,AJAX请求/index.php?app=resource&mod=test&act=212,则会指向到
    • Linux环境下的/var/www/Module/HelloWorld/data_test_212.json文件
    • Windows环境:进入D:\\www\\Module\\HelloWorld\\data_test_212.json文件
到这里，Smarty模板+AJAX页面就建成了!前端调试后端数据就是这么轻松!

发现C.F.F更多的特性

  • 使用页面调用Component目录下的公共组件,使用Smarty的include语法既可。C.F.F会连同公共组件中的Mock数据/JavaScript/CSS一同加载。
    • 无需再次Mock数据，及复制粘贴JavaScript/CSS目录，在主页面调用多个公共组件会更加方便
    • 调用者无需了解被调用者的业务、数据，只需一行代码完全集成
  • 公共组件开发者在Component目录下index.html也可以写入一些demo代码用于调试，,为公共组件开发尽可能的还原真实场景。集成打包时会自动过滤这些调试代码。 
    访问页面http://127.0.0.1:8006/Component/目录名xxx/,实现了公共组件开发既调试，开发既Demo！
  • C.F.F是支持分级路由的，有效的避免了协同开发中路由配置文件被占用的情况。如果项目不需要分级路由，只需要在根目录下建一个route.json即可。
